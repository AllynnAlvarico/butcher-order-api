How to set up the Project
===========================

1. Clone the Repository
    - git clone https://github.com/your-user/order-api-butcher.git
    - cd order-api-butcher
2. Configure Database
    - Create the MySQL database using the provided SQL script located in the 'sql_script' folder.
        - Open your MySQL client or workbench.
        - Execute the SQL script to create the necessary database and tables.
            - Create the database: use the "Create Database" script.
            - Create the tables: use the "Create Tables" script.
                - Create or use the "Create Customer Table" script. (First)
                - Create or use the "Create Product Table" script. (Second)
                - Create or use the "Create Orders Table" script. (Third)
                - Create or use the "Create Order Items Table" script. (Fourth)
            -Insert sample data: use the "Insert Sample Data" script.
                - Use the "Insert Values into Customer Table" script.
                - Use the "Insert Values into Product Table" script.
                - Note: Orders and Order Items tables will be populated through the API.
3. Update Database Configuration
    - Open the 'src/main/resources/application.properties' file.
    - Update the database URL, username, and password to match your MySQL configuration.
        - spring.datasource.url=jdbc:mysql://localhost:3306/your_database_name
        - spring.datasource.username=your_username
        - spring.datasource.password=your_password

4. Run the Application for Development
    - Hit "Run" in your IDE (e.g., IntelliJ, Eclipse) to start the Spring Boot application.
    - Go to http://localhost:8080 in your web browser to access the API.
5. Test the API Endpoints
    - /api/customers
        - Manage customers
            - Retrieve all customers (endpoint: GET /api/customers/all)
            - Retrieve a customer by ID (endpoint: GET /api/customers/{id})
            - Create a new customer (endpoint: POST /api/customers/register)
            - Delete a customer (endpoint: DELETE /api/customers/{id})
